And so ... it looks like this quick-and-dirty network is actually capable of detecting real bat calls. Kind of encouraging !¶

What was done :¶

The 0.5 second audio recording was split into non-overlapping 10millisecond segments. The spectrograms for each of these segments were calculated with 64 sample FFT window, 32 sample overlap. This gave rise to an image-like matrix of 33 x 59 dimensions. This was fed into the pre-trained network and the prediction confidence was plotted over the spectrogram itself.

Concluding remarks : The bat call detector is not fantastic, buhuuuut - it does its job fairly okay, given the kind of training data it got. The training data was basically just FM chirps to show it what bat calls are like, and a few other sounds (white noise, upward ultrasonic chirps, downward audible chirps) to show it what bat calls are not like....and it's already able to predict the presence of a bat call over real recordings.¶

Next steps :¶

  • 1 : Increase the size + variation in the training data, consider multiharmonic bat calls, consider adding reverberation
  • 2 : Instead of using non-overlapping audio segments to get presence/absence predictions, try using overlapping audio segments, this might generate a more continuous prediction of where the bat calls are in the audio data !
  • 3 : Of course, ultimately it's best to have the network learn from real data that's been labelled manually, but to me this feels very cumbersome,given that I've already managed so much with purely synthesised data !!!
